# Support

## Getting Help

If you need help using or developing LumaSync:

1. Read `README.md` for setup and scripts.
2. Check `docs/debugging.md` for troubleshooting flow.
3. Search existing issues before opening a new one.

## When Opening a Support Issue

Include the following to speed up triage:

- What you are trying to do
- What you expected
- What happened instead
- OS and environment details
- Relevant logs and screenshots

## Security Issues

For suspected vulnerabilities, do not open a public issue.
Use the private reporting path described in `SECURITY.md`.
